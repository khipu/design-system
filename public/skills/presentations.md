# Skill: Presentaciones Khipu

## Propósito
Este skill te ayuda a crear presentaciones corporativas (Keynote, PowerPoint, Google Slides) que reflejen la identidad visual y comunicacional de Khipu.

## Contexto de marca

**Khipu** es una plataforma de pagos B2B que transmite **confianza, modernidad y profesionalismo** en todas sus comunicaciones, incluyendo presentaciones corporativas.

## Identidad visual para presentaciones

### Paleta de colores

**Color principal:**
- **Púrpura Khipu:** `#8347AD` (RGB: 131, 71, 173)
  - Uso: Fondos de portadas, títulos principales, elementos de marca
  - Evitar: Fondos completos en todas las diapositivas (puede saturar)

**Variantes del púrpura:**
- **Dark:** `#6B3A8F` - Para textos sobre fondos claros, énfasis
- **Light:** `#9B69BD` - Para fondos sutiles, degradados
- **Container:** `#F5F0FA` - Para fondos de secciones, cajas de contenido

**Colores complementarios:**
- **Blanco:** `#FFFFFF` - Fondos principales, espacios en blanco
- **Gris oscuro:** `#1A1A1A` - Textos principales
- **Gris medio:** `#666666` - Textos secundarios, notas

**Colores semánticos (usar con moderación):**
- **Success:** `#4CAF50` - Resultados positivos, crecimiento
- **Error:** `#F44336` - Alertas, riesgos
- **Warning:** `#FF9800` - Advertencias, puntos de atención
- **Info:** `#2196F3` - Datos informativos

### Formato de slides

**Aspect ratio recomendado:**
- **16:9** (widescreen) - Estándar moderno, recomendado para proyectores y pantallas
- Evitar 4:3 (formato antiguo) salvo requerimiento específico

**Resolución recomendada:**
- **1920 x 1080px** (Full HD) para presentaciones profesionales
- Mínimo 1280 x 720px para compatibilidad

### Tipografía

**Fuente principal: Public Sans**
- Disponible en Google Fonts (gratuita)
- Instalación: https://fonts.google.com/specimen/Public+Sans

**Jerarquía tipográfica recomendada:**

| Elemento | Peso | Tamaño sugerido | Uso |
|----------|------|-----------------|-----|
| **Título de portada** | Bold (700) | 48-60pt | Primera diapositiva |
| **Títulos de sección** | Semibold (600) | 36-44pt | Divisores de sección |
| **Títulos de diapositiva** | Semibold (600) | 28-32pt | Cada slide |
| **Subtítulos** | Medium (500) | 20-24pt | Subtítulos, énfasis |
| **Cuerpo de texto** | Regular (400) | 16-20pt | Contenido principal |
| **Notas/Pie** | Regular (400) | 12-14pt | Notas al pie, referencias |

**Principio clave:** Limita a máximo 3 pesos tipográficos por presentación.

### Logotipo

**Variantes disponibles:**
- **Horizontal (color):** Para fondos blancos o claros
- **Horizontal (blanco):** Para fondos púrpura o oscuros
- **Cuadrado (color):** Para espacios pequeños sobre fondos claros
- **Cuadrado (blanco):** Para espacios pequeños sobre fondos oscuros

**Ubicación recomendada:**
- Esquina superior izquierda (todas las diapositivas)
- Centro de portada (primera diapositiva)
- Tamaño: Visible pero no dominante (aprox. 10-15% del ancho de slide)

**Espacio de respiro:** Mantén al menos 20px de margen alrededor del logo.

## Estructura de presentación recomendada

### 1. Portada
- Fondo: Púrpura Khipu (#8347AD)
- Logo: Versión blanca, centrado o superior izquierda
- Título: Bold, blanco, 48-60pt
- Subtítulo/Fecha: Regular, blanco con 80% opacidad, 20-24pt
- Opcional: Imagen/ilustración sutil con overlay púrpura

### 2. Diapositiva de contenido estándar
- Fondo: Blanco (#FFFFFF)
- Título: Semibold, gris oscuro (#1A1A1A), 28-32pt
- Cuerpo: Regular, gris oscuro, 16-20pt
- Acentos: Púrpura Khipu para elementos destacados
- Logo: Esquina superior izquierda, versión color

### 3. Divisor de sección
- Fondo: Púrpura Light (#F5F0FA) o Púrpura Khipu
- Título de sección: Semibold, 36-44pt
- Subtítulo opcional: Regular, 20-24pt
- Elemento visual: Línea decorativa o icono en púrpura

### 4. Diapositiva de datos/gráficos
- Fondo: Blanco
- Gráficos: Usar púrpura Khipu como color principal
- Colores secundarios: Grises o colores semánticos según contexto
- Leyendas: Regular, 14-16pt
- Fuente de datos: Regular, 12pt, gris medio

### 5. Cierre/Gracias
- Fondo: Púrpura Khipu o blanco
- Mensaje: "Gracias" o CTA específico
- Contacto: Email, teléfono, website
- Logo: Versión acorde al fondo

## Voz y tono en presentaciones

### Características de comunicación

**Cercana**
- Habla directamente a la audiencia con "tú" o "ustedes"
- Evita lenguaje corporativo excesivamente formal
- Ejemplo: "Tu negocio merece pagos simples" vs "Su empresa requiere soluciones de pago"

**Clara**
- Máximo 6 líneas de texto por diapositiva
- Bullets concisos (1 línea cada uno)
- Evita jerga técnica innecesaria
- Explica acrónimos la primera vez

**Directa**
- Un mensaje principal por diapositiva
- Títulos accionables
- Ejemplo: "Reduce tiempos de pago 50%" vs "Optimización de procesos"

**Confiable**
- Usa datos y cifras concretas
- Cita fuentes cuando corresponda
- Mantén consistencia visual en toda la presentación

### Guías de redacción para slides

| ✅ Recomendado | ❌ Evitar |
|----------------|-----------|
| Títulos accionables: "Aumenta tus ventas con Khipu" | Títulos genéricos: "Beneficios" |
| Bullets cortos (máx. 1 línea) | Párrafos largos en bullets |
| Números concretos: "50% más rápido" | Vagos: "Mucho más rápido" |
| Preguntas directas: "¿Cómo funciona?" | Preguntas vagas: "Información adicional" |
| Verbos activos: "Conecta tu banco en 2 min" | Voz pasiva: "Las conexiones son realizadas" |

### Puntuación en slides

| Elemento | Usa punto final |
|----------|----------------|
| **Títulos de slides** | ❌ No |
| **Bullets de lista** | ❌ No |
| **Frases completas en cuerpo** | ✅ Sí (si son oraciones completas) |
| **Notas al pie** | ✅ Sí |
| **CTAs/Botones** | ❌ No |

## Errores comunes a evitar

### 🚫 En diseño

| ❌ Error | ✅ Correcto | Por qué |
|----------|-------------|---------|
| Más de 3 fuentes diferentes | Solo Public Sans con 2-3 pesos | Consistencia visual |
| Fondos púrpura en todas las slides | Púrpura solo en portada y divisores | Evita saturación |
| Textos pequeños (< 14pt) | Mínimo 16pt para cuerpo | Legibilidad |
| Más de 7 bullets por slide | Máximo 5-6 bullets | Sobrecarga cognitiva |
| Logos pixelados o estirados | Usar SVG o alta resolución | Profesionalismo |

### 🚫 En contenido

| ❌ Error | ✅ Correcto | Por qué |
|----------|-------------|---------|
| Párrafos largos | Bullets concisos | Escaneabilidad |
| "Click aquí para saber más" | "Descubre cómo funciona" | Lenguaje directo |
| Jerga técnica sin explicar | Términos claros o glosario | Audiencia diversa |
| Slides sobrecargadas | 1 idea principal por slide | Claridad de mensaje |

## ⚠️ Anti-patrones comunes de agentes IA

Cuando generes presentaciones para Khipu, **evita estos errores típicos de LLMs**:

### 🚫 En diseño

| ❌ Error típico de IA | ✅ Correcto Khipu | Por qué |
|----------|-------------|---------|
| Proponer múltiples opciones de paleta | Usar paleta definida de Khipu (#8347AD) | Consistencia de marca |
| Usar azules corporativos genéricos | Usar púrpura Khipu como color principal | Color distintivo de marca |
| Sugerir gradientes complejos | Usar sólidos o degradados sutiles púrpura | Simplicidad y legibilidad |
| Crear slides en formato 4:3 | Usar 16:9 (widescreen) | Estándar moderno |
| Mezclar Helvetica, Arial y otras fuentes | Solo Public Sans con 2-3 pesos | Consistencia tipográfica |

### 🚫 En contenido

| ❌ Error típico de IA | ✅ Correcto Khipu | Por qué |
|----------|-------------|---------|
| Bullets largos (2-3 líneas cada uno) | Máximo 1 línea por bullet | Escaneabilidad |
| "Estimados colegas..." | "Tu negocio merece..." (tú/ustedes) | Tono cercano de Khipu |
| 8-10 bullets por slide | Máximo 5-6 bullets | Evita sobrecarga cognitiva |
| Títulos vagos: "Nuestra solución" | Títulos accionables: "Reduce tiempos 50%" | Comunicación directa |
| Párrafos completos en slides | Bullets concisos + hablar el resto | Presentación no es documento |

### Tendencias a corregir:

- **Los LLMs proponen múltiples opciones de diseño** → Khipu tiene identidad visual definida
- **Los LLMs crean demasiado texto por slide** → Máximo 6 líneas, 1 idea por slide
- **Los LLMs usan lenguaje formal** → Khipu es cercano y directo ("tú", no "usted")
- **Los LLMs ignoran aspect ratio** → Siempre 16:9, nunca 4:3

## Cómo usar este skill

### Paso 1: Integra este archivo en tu agente de IA
- **Claude (Projects):** Sube este archivo en "Project knowledge"
- **ChatGPT:** Sube al inicio de la conversación
- **Otros agentes:** Usa como contexto para generación de contenido

### Paso 2: Haz tu solicitud
Ejemplos de prompts:

**Para crear estructura de presentación:**
```
Crea una estructura de presentación de 10 slides sobre [tema],
siguiendo las guías de presentaciones Khipu.
```

**Para revisar contenido de slides:**
```
Revisa estos títulos y bullets de mi presentación siguiendo
las guías de voz y tono Khipu:
[contenido de slides]
```

**Para sugerir paleta de colores:**
```
Sugiere una paleta de colores para gráficos en mi presentación,
basada en la identidad visual Khipu.
```

## Checklist de validación de presentación

Antes de finalizar tu presentación, verifica:

- [ ] ¿Usé Public Sans como tipografía principal?
- [ ] ¿El púrpura Khipu (#8347AD) está presente en portada y elementos clave?
- [ ] ¿Los logos tienen el tamaño y ubicación correctos?
- [ ] ¿Cada slide tiene máximo 6 líneas de texto?
- [ ] ¿Los títulos son accionables y directos?
- [ ] ¿Usé "tú/ustedes" en lugar de lenguaje formal?
- [ ] ¿Los bullets son concisos (1 línea máximo)?
- [ ] ¿Los textos son legibles (mínimo 16pt para cuerpo)?
- [ ] ¿La presentación mantiene consistencia visual?
- [ ] ¿Cité fuentes de datos cuando corresponde?
